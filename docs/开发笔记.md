# 在线聊天室 - 开发笔记

## 一、环境搭建阶段

### 问题1：MySQL 安装后无法在命令行使用

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 输入 mysql -u root -p 提示"不是内部或外部命令" |
| 分析问题 | MySQL未添加到系统PATH环境变量 |
| 解决问题 | 将 C:\Program Files\MySQL\MySQL Server 8.0\bin 添加到PATH |

### 问题2：Eventlet 弃用警告

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 运行 app.py 时出现 EventletDeprecationWarning |
| 分析问题 | Eventlet 已被官方标记为弃用 |
| 解决问题 | 警告不影响运行，暂时忽略；未来可迁移到 asyncio 版本 |

## 二、后端开发阶段

### 问题3：密码加密方式的选择

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 不确定用哪种加密方式最安全 |
| 分析问题 | 需要不可逆哈希加盐算法，不能用MD5/SHA1 |
| 解决问题 | 选择 bcrypt，注册用 hashpw，登录用 checkpw |

### 问题4：WebSocket 消息只能自己看到

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 用户发送消息后只有自己能收到 |
| 分析问题 | emit() 默认只发送给当前客户端 |
| 解决问题 | 添加 broadcast=True 参数实现广播 |

### 问题5：Session 在 WebSocket 事件中无法获取

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | handle_message 中用 session.get 返回 None |
| 分析问题 | WebSocket 事件中默认无法直接访问 Flask session |
| 解决问题 | 确保登录时正确设置 session，前端连接时携带 Cookie |

## 三、前端开发阶段

### 问题6：页面加载后 WebSocket 连接失败

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 浏览器控制台报 WebSocket connection failed |
| 分析问题 | 页面加载时就连接 WebSocket，但用户尚未登录 |
| 解决问题 | 将 initSocket() 放在登录成功之后执行 |

### 问题7：历史消息重复加载

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 登录后历史消息出现重复 |
| 分析问题 | loadHistoryMessages() 被多次调用 |
| 解决问题 | 检查调用位置，确保只调用一次 |

### 问题8：消息列表不自动滚动

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 新消息出现时需要手动滚动 |
| 分析问题 | 添加消息后滚动条位置不变 |
| 解决问题 | 调用 scrollIntoView() 自动滚动到底部 |

## 四、测试阶段

### 问题9：同一浏览器无法测试多人聊天

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 注册第二个账号会覆盖第一个的登录状态 |
| 分析问题 | Cookie和Session在同一浏览器中共享 |
| 解决问题 | 使用不同浏览器或浏览器无痕模式 |

### 问题10：消息时间显示不准确

| 项目 | 内容 |
| --- | --- |
| 遇到问题 | 消息时间与实际时间不符 |
| 分析问题 | MySQL时区与Python时区可能不一致 |
| 解决问题 | 前端只显示时分秒，不显示日期 |
